⚠️ 重要说明：Cloudflare Pages 的局限性
Cloudflare Pages 是什么？
Cloudflare Pages 是一个静态网站托管平台，主要功能：

托管 HTML、CSS、JavaScript 等静态文件
部署前端框架项目（Vue、React、Next.js 等）
不支持运行 Java、Spring Boot 等后端服务
你的项目架构分析
qa-platform（前后端集成）
├── src/                    ← Java/Spring Boot 后端代码
├── target/                 ← 编译后的 jar 包
├── pom.xml                 ← Maven 配置
└── uploads/                ← 上传文件存储
这是一个典型的 Java 后端项目，无法直接在 Cloudflare Pages 上运行！

✅ 正确的部署方案
方案一：Spring Boot 项目整体部署到 Railway（推荐⭐）
Railway 是一个支持 Java 的免费部署平台，可以一键部署 Spring Boot 项目。

部署架构：
┌─────────────────┐      ┌─────────────────┐
│   Railway       │      │   Railway       │
│   (后端服务)    │◄────►│   (MySQL)       │
│   qa-platform  │      │   数据库         │
└─────────────────┘      └─────────────────┘
        │
        │ jar 包运行
        ▼
用户通过 your-app.railway.app 访问
📋 完整操作步骤
步骤 1：在 GitHub 创建仓库（已有可跳过）

1.
登录 GitHub
2.
创建新仓库 qa-platform
3.
将项目代码上传（参考之前 IDEA 上传教程）
步骤 2：修改数据库配置

由于本地 MySQL 无法远程访问，需要修改为云数据库。推荐使用：

Railway MySQL（与后端同平台，推荐）
PlanetScale（免费 MySQL）
Neon（免费 PostgreSQL，可与 MySQL 兼容）
创建 E:\Test\XiangMu\LunTan_Demo\qa-platform\src\main\resources\application-prod.yml：

yaml
spring:
  datasource:
    url: ${DB_URL}           # Railway 环境变量
    username: ${DB_USERNAME}
    password: ${DB_PASSWORD}
    driver-class-name: com.mysql.cj.jdbc.Driver
  jpa:
    hibernate:
      ddl-auto: none          # 生产环境不自动创建表
    show-sql: false

server:
  port: ${PORT:8080}
步骤 3：修改 pom.xml 支持打包

在 pom.xml 中添加配置：

xml
<build>
    <plugins>
        <plugin>
            <groupId>org.springframework.boot</groupId>
            <artifactId>spring-boot-maven-plugin</artifactId>
            <!-- 排除 provided 依赖 -->
            <configuration>
                <excludes>
                    <exclude>
                        <groupId>org.apache.tomcat.embed</groupId>
                        <artifactId>tomcat-embed-jasper</artifactId>
                    </exclude>
                </excludes>
            </configuration>
        </plugin>
    </plugins>
</build>
步骤 4：创建 Railway 项目

1.
访问 railway.app 并登录
2.
点击 "New Project" → "Deploy from GitHub repo"
3.
选择你的 qa-platform 仓库
4.
Railway 会自动检测为 Java 项目
步骤 5：配置环境变量

在 Railway 项目设置中添加：

变量名	值	说明
DB_HOST	MySQL 主机地址	Railway MySQL 提供
DB_PORT	3306	MySQL 端口
DB_NAME	qa_platform	数据库名
DB_USERNAME	root	用户名
DB_PASSWORD	xxx	密码
PORT	8080	Railway 端口
步骤 6：配置构建命令

Railway 会自动检测 Maven 项目，如需手动配置：

bash
mvn clean package -DskipTests
启动命令：

bash
java -jar target/qa-platform-1.0-SNAPSHOT.jar

5.

6.
分别部署
